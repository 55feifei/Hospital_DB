## 4. 数据访问层设计

### 4.1 连接管理（src/db.py）

- 每次访问按需建立 PyMySQL 连接，操作完毕即关闭
- 提供三个上下文：
  - `Database.cursor(commit=False)`：普通查询/写入，可选自动提交
  - `Database.transaction()`：显式事务，异常自动回滚
  - `Database.call_proc(name, args)`：调用存储过程并取出 OUT 参数

### 4.2 调用存储过程的细节

PyMySQL 的 `cursor.callproc()` 把 OUT 参数存成 `@_proc_n` 形式的会话变量。本系统封装：
1. 把 `(IN, IN, OUT, OUT)` 全部当作普通参数传入（OUT 用 0 / '' 占位）
2. 通过 `SELECT @_sp_book_appointment_2, @_sp_book_appointment_3` 取回真实 OUT 值
3. 同时收集 `cur.fetchall() + cur.nextset()` 形成多结果集列表

## 5. 事务与并发控制

| 场景             | 控制手段                                         |
|------------------|--------------------------------------------------|
| 预约抢号         | `SELECT … FOR UPDATE` 行锁排班记录               |
| 取消预约         | 行锁 + 状态机校验 + 事务回滚                     |
| DDL 变更         | `IF EXISTS / IF NOT EXISTS`，幂等                |
| 触发器内异常     | `SIGNAL SQLSTATE '45000'` 抛错并自动回滚         |

并发测试示例（两个客户端同时抢最后一号）：
```
Session A: CALL sp_book_appointment(1, 100, @a, @msg);  -- 拿到行锁
Session B: CALL sp_book_appointment(2, 100, @a, @msg);  -- 阻塞等待
Session A: COMMIT;
Session B: 唤醒后 SELECT remaining → 0 → 返回"号源已满"
```

## 6. UI 设计要点

### 6.1 风格规范
- 使用 Fusion 主题统一外观
- 主操作按钮使用强色（绿色 #27ae60 = 成功操作；红色 #e74c3c = 危险操作；橙色 #f39c12 = 财务操作；蓝色 #3498db = 信息操作）
- 表格统一使用 `Stretch` 自动填充列宽
- 中文按钮文字 + 中文表头，符合医院实际使用习惯

### 6.2 输入校验（双层）
- **客户端**：
  - 用户名：`^[A-Za-z0-9_]{3,30}$`
  - 手机号：`^1[3-9]\d{9}$`
  - 身份证：`^\d{17}[\dXx]$`
- **服务端**：
  - DB 层 CHECK / UNIQUE / FK 约束兜底
  - 触发器抛出业务异常

### 6.3 错误反馈
- 数据库异常 → `QMessageBox.critical`
- 业务校验失败 → `QMessageBox.warning`
- 操作成功 → `QMessageBox.information`
- 危险操作 → `QMessageBox.question` 二次确认

## 7. 部署与运行

### 7.1 环境
- Python 3.9+ / pip
- MySQL 8.0+ 已启动
- Windows 10/11 或 Linux

### 7.2 一键初始化（首次部署）

```bash
# 1. 安装 Python 依赖
pip install -r requirements.txt

# 2. 修改 src/config.py 中 password

# 3. 初始化 DB（按顺序）
mysql -u root -p < sql/01_schema.sql
mysql -u root -p < sql/02_triggers.sql
mysql -u root -p < sql/03_procedures.sql
mysql -u root -p < sql/04_views.sql
mysql -u root -p < sql/05_init_data.sql
mysql -u root -p < sql/06_security.sql

# 4. 启动
python src/main.py
```

### 7.3 升级与维护
- DDL 变更：通过新增 `sql/08_migration_xxx.sql` 增量脚本
- 备份恢复：`sql/07_backup_restore.md` 给出操作流程

## 8. 异常与日志

- DB 异常：PyMySQL 抛出 `IntegrityError / OperationalError` → UI 层 `QMessageBox.critical` 给出原始信息
- 业务异常：通过存储过程 OUT 参数 `p_msg` 传出（如"号源已满"），UI 显示 `warning`
- 关键操作（登录、爽约清理）追加 `last_login` / `remark` 字段留痕

## 9. 扩展点

| 扩展场景                | 实现思路                                    |
|-------------------------|---------------------------------------------|
| Web 端                  | 把 dao/service 层不动，UI 替换为 Flask API + Vue |
| 微信通知                | 加事件表 `notification`，由 trigger 写入；外部消费者推送 |
| 多医院                  | 各表加 `hospital_id` 列，外键到 `hospital`  |
| 排队叫号大屏            | 订阅 `appointment` 状态变化（CDC/Redis 通知） |
