# 医院门诊预约管理系统 — 操作手册

## 一、安装与初始化

### 1.1 系统要求
- 操作系统：Windows 10/11 或 Linux（Ubuntu 22.04 验证通过）
- Python 3.9 +
- MySQL 8.0 +

### 1.2 安装步骤

```bash
# 1. 克隆 / 解压项目
cd f:\数据库大作业2026

# 2. 安装 Python 依赖
pip install -r requirements.txt

# 3. 修改 src/config.py 中的 password 为你本机 MySQL root 密码

# 4. 按顺序执行 SQL 脚本
mysql -u root -p < sql/01_schema.sql
mysql -u root -p < sql/02_triggers.sql
mysql -u root -p < sql/03_procedures.sql
mysql -u root -p < sql/04_views.sql
mysql -u root -p < sql/05_init_data.sql
mysql -u root -p < sql/06_security.sql

# 5. 启动程序
python src/main.py
```

## 二、患者操作流程

### 2.1 预约挂号
1. 双击桌面图标启动程序，进入登录页
2. 输入患者账号（如 pat001）和密码 123456 → 登录
3. 默认进入"预约挂号"标签
4. 在科室下拉框中选择科室（如"内科"）
5. 在"起始日期"中选择查询日期，下方表格自动显示该科室未来排班
6. 在表格中**单击选中一行**剩余号源 > 0 的排班
7. 点击下方绿色"预约选中排班"按钮
8. 弹出"预约成功，排队序号 N"提示

### 2.2 查看 / 取消预约 / 缴费
1. 切换到"我的预约"标签
2. 表格列出本人所有历史预约
3. 选中一条状态为"已预约"的记录
4. 点击红色"取消选中预约"按钮 → 二次确认 → 取消成功
5. 点击橙色"缴费"按钮 → 状态变为"已支付"

### 2.3 查看病历
1. 切换到"就诊记录"标签
2. 表格显示历次就诊的诊断、处方等

## 三、医生操作流程

### 3.1 查看当日预约
1. 用医生账号（如 doc001）登录
2. 工作台顶部显示 医生姓名、职称、科室
3. 默认显示当日预约列表（按序号排序）
4. 切换上方"查询日期"可查看其它日期的预约

### 3.2 完成就诊
1. 选中一行状态为"已预约"的记录
2. 点击"完成就诊（写病历）"按钮
3. 在弹出的病历对话框中填写：
   - 主诉（必填）
   - 诊断（必填）
   - 处方
   - 医嘱
4. 点击"保存"
5. 该行状态自动变为"已就诊"，触发器自动建立病历记录

### 3.3 编辑历史病历
1. 选中一行已就诊预约
2. 点击"查看/编辑病历"
3. 在对话框中修改后保存

## 四、管理员操作流程

### 4.1 科室管理
- 标签 1 "科室管理" → 新增 / 修改 / 删除
- 删除存在医生引用的科室会被外键约束阻止

### 4.2 医生管理
- 标签 2 "医生管理" → 新增 / 修改 / 删除
- 删除存在未完成预约的医生会被触发器阻止

### 4.3 患者管理
- 标签 3 "患者管理" → 顶部搜索框支持按姓名/手机/身份证模糊搜索
- 校验：身份证 18 位、手机号 1[3-9]开头 11 位

### 4.4 排班管理
- 标签 4 "排班管理"
- 顶部下拉选择医生 → 表格显示其所有排班
- "新增排班"：选择日期、时段（上午/下午）、号源数
- "停诊/恢复"：切换状态，停诊后患者无法预约该排班
- "删除"：仅限无预约的排班可删

### 4.5 全部预约
- 标签 5 "全部预约"
- 显示最近 500 条预约记录

### 4.6 统计与系统
- 标签 6 "统计/系统"
- "调用 sp_daily_statistics（游标）"：选指定日期 → 按科室统计预约数 / 已就诊 / 收入
- "一键标记爽约（sp_mark_no_show）"：把指定日期之前仍处于"已预约"的记录批量标记为"爽约"
- 中部"医生工作量"：来自 v_doctor_workload 视图
- 底部"最忙医生"：通过子查询找出预约总数最多的医生

## 五、常见问题

### Q1：登录后弹"无法连接数据库"
- 检查 MySQL 服务是否启动
- 检查 src/config.py 里的 password 是否正确
- Windows 防火墙是否拦截 3306 端口

### Q2：预约时弹"号源已满"
- 该排班的剩余号源为 0，触发器拒绝
- 改选其它时段或日期

### Q3：删除医生失败"该医生存在未完成预约"
- 触发器 `trg_doctor_before_delete` 阻止
- 先取消该医生的所有未完成预约或将其状态改为"离职"

### Q4：忘记 root 密码 / 数据库账号
- 重置 root 密码：参考 MySQL 官方手册 `--skip-grant-tables`
- 应用账号 `hospital_app / App@2026` 在 `sql/06_security.sql` 中可改

### Q5：怎么备份 / 还原？
- 见 [`sql/07_backup_restore.md`](../sql/07_backup_restore.md)
